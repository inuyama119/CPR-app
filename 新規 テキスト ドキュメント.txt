# 胸骨圧迫解析ロジック 仕様レポート

本プロジェクト（Bystander CPR Trainer）における、AIによる姿勢およびリズムの解析ロジックの最新仕様をまとめました。

---

## 1. 姿勢判定（垂直性の解析）

「腕が床に対して垂直に保たれているか」を、肩と手首の座標から算出します。

| 項目 | 内容 |
| :--- | :--- |
| **基準点** | 肩（Shoulder）と手首（Wrist）を結ぶベクトル |
| **角度計算** | `Math.atan2(dx, dy)` を使用。垂直状態を `0度` と定義 |
| **補正** | 画面のアスペクト比（横幅・高さ）を考慮したピクセル単位の計算 |
| **合格閾値** | **20度以内**（垂直から左右20度までの傾きを許容） |
| **判定ロジック** | 以下の2条件を同時に満たす場合に「合格」とする<br>1. 全計測期間の角度の**中央値（Median）**が20度以内<br>2. 最も深く押し込んだ瞬間（Peak）の角度が20度以内 |

---

## 2. リズム判定（BPMの解析）

「適切なテンポで圧迫が行われているか」を、手首の上下動から算出します。

| 項目 | 内容 |
| :--- | :--- |
| **計測手法** | 手首のY座標のピーク（最深部から戻る瞬間）を検出し、その間隔を算出 |
| **合格範囲** | **100 〜 120 BPM** （JRCガイドライン準拠） |
| **スムージング** | 指数移動平均（EMA, Factor: 0.4）により、センサーノイズを抑制 |
| **リアルタイム表示** | 直近3回の圧迫間隔の平均値を画面に表示 |

---

## 3. 解析ライフサイクル管理

データの整合性を保つため、以下のフローで制御しています。

1. **初期化（Start）**:
* 計測開始ボタンが押された瞬間に、過去の履歴データをすべて破棄します。
2. **データ蓄積（Measuring）**:
* MediaPipe Poseから取得した座標を毎フレーム解析し、履歴に保存します。
3. **判定（Finish）**:
* 計測終了後、**即座に計算（calculateResult）**を実行してからリセットを行います。

---

## 4. 総合評価（ランク）システム

| ランク | 判定条件 | ユーザーへのメッセージ |
| :---: | :--- | :--- |
| **◎** | 姿勢とリズムが**両方合格** | 完璧です！素晴らしい！ |
| **○** | 姿勢またはリズムの**どちらか一方が合格** | あと一歩です！ |
| **△** | 姿勢とリズムが**両方不合格** | 練習あるのみ！ |

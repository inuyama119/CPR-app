# 胸骨圧迫マスター (CPR Evaluation) アプリ仕様書

## 1. アプリケーション概要
- **目的**: スマートフォンのカメラ等を用いて胸骨圧迫（CPR）のトレーニングと、AIによるフォーム・リズム解析を行うWebアプリケーション。
- **対象**: CPR（心肺蘇生法）の練習を行いたい一般ユーザーおよび講習受講者。
- **準拠ガイドライン**: JRC蘇生ガイドライン 2020
- **主要技術スタック**:
  - **フレームワーク**: React (Vite環境)
  - **ルーティング**: `react-router-dom` (HashRouter使用、GitHub Pages対応のため)
  - **AI解析エンジン**: MediaPipe Pose Landmarker (`@mediapipe/tasks-vision`)
  - **スタイリング**: Vanilla CSS (`index.css` にて独自変数を定義)
  - **アイコン・アニメーション**: `lucide-react`, `framer-motion`, `canvas-confetti`
  - **デプロイ環境**: GitHub Pages (自動CI/CD: GitHub Actions `.github/workflows/deploy.yml`)
  - **アクセス解析**: Google Analytics (GA4)

---

## 2. モードと機能一覧

### 2.1 ホーム画面 (`Home.jsx`)
- **役割**: アプリの起動画面。各モードへの遷移と、外部アンケートへのリンクを提供する。
- **UI特徴**: `framer-motion` を使用したバウンスアニメーションと、鼓動を模した `pulse` アニメーションのアイコンを配置。

### 2.2 姿勢復習モード (`Training.jsx`)
- **役割**: 動画解析は行わず、正しい姿勢とリズムを直感的に復習するためのトレーニング専用モード。
- **機能（メトロノーム）**:
  - `110 BPM` のテンポで、定期的なビープ音（Web Audio API を使用: 880Hz, 0.05秒間）を再生し、理想的な圧迫リズムをユーザーに体感させる。
- **機能（ポイント表示）**:
  - 「腕を垂直に」「手の付け根で」「5cm沈む強さ」という3つの重要ポイントとイラストを表示。

### 2.3 ビデオで評価モード (`VideoEvaluation.jsx`)
- **役割**: スマホカメラ等で撮影したユーザーの胸骨圧迫動画をAI（MediaPipe）を用いて解析し、スコアデータを算出するメインモード。
- **状態遷移とUI制御**:
  1. **プリロード**: アプリ起動時（`App.jsx`）にバックグラウンドでAIモデルを非同期ダウンロードし、待ち時間を削減。
  2. **カメラ撮影/動画選択**: `<input type="file" accept="video/*" capture="environment">` により、端末のネイティブカメラを起動。
  3. **安全装置（待機UI）**: 動画選択後、AIモデルのダウンロードが未完了、または動画の解像度が計算されていない（`videoWidth === 0`）場合は、動画プレイヤー上に黒い半透明の「AIモデルを準備中...」オーバーレイを表示し、不完全な状態での解析開始を物理的にブロックする。

## 3. AI動画解析ロジック詳細

解析は `requestAnimationFrame` ループ（`predictLoop` 関数）内で行われ、毎フレームごとに `globalPoseLandmarker.detectForVideo()` が呼ばれます。

- **解析モデル**: `pose_landmarker_full.task` (GPU Delegate モード)
- **解析除外フレーム**: 動画の最初と最後の3秒間は、体勢を整えたりスマホの録画停止ボタンを押す時間とみなし、解析計算から除外する。
  `if (videoTime < 3 || videoTime > video.duration - 3) return;`

### 3.1 骨格座標（ランドマーク）の取得
カメラに対して横向きでCPRを行うことを前提としているため、カメラに最もよく写っている（信頼度が高い）側の腕と肩を自動選択する。
- **手首 (`wrist`)**: 左右の手首（Landmark 15 と 16）のうち、`visibility`（検出信頼度）が高い方を採用。
- **肩 (`shoulder`)**: 左右の肩（Landmark 11 と 12）のうち、`visibility` が高い方を採用。

### 3.2 傾きの算出（姿勢チェック）
基準となる肩と手首の X, Y 座標の差分（`dx`, `dy`）から、アークタンジェント (`Math.atan2`) を用いて地面に対する垂直からのズレ角度を計算する。
- **計算式**: `const currentTilt = Math.abs(Math.atan2(dx, dy) * 180 / Math.PI);`
- 各圧迫ごとの傾きを配列 `tiltValues` に保存し、最終的にその **中央値 (Median)** をユーザーの評価値とする。

### 3.3 圧迫回数とBPMの算出（リズムチェック）
手首の Y座標（画面上下方向の動き）の変位を監視し、圧迫の往復運動を検知する。

1. **スムージング**: 細かいノイズや手ブレを吸収するため、指数移動平均を適用。
   `smoothedY = 0.4 * wrist.y + 0.6 * lastY`
2. **押し下げ判定**: Y座標の差分（`diff`）が `0.003` より大きい場合、体が沈み込んでいる（`isMovingDown = true`）と判定。
3. **押し上げ（1回の圧迫完了）判定**: `isMovingDown` が true の状態で、`diff < -0.001`（体が上がり始めた）になった瞬間を「1回の圧迫」としてカウント (`compressions++`)。
4. **BPM計算**: 
   - 前回の圧迫完了時刻と今回の時刻の差分（`interval` ミリ秒）から瞬間BPMを計算。
   - **計算式**: `60000 / interval`
   - ※チャタリング（微細な揺れの二重カウント）防止のため、`interval > 400` ミリ秒（= 150BPM以上は異常値）の制限を設ける。
   - 各圧迫ごとのBPMを配列 `bpmValues` に保存し、最終的にその **中央値 (Median)** をユーザーの評価値とする。

## 4. 診断とスコアリング (`Result.jsx`)

算出された `medianBpm` と `medianTilt` をベースに、JRCガイドラインに沿ったフィードバックを行う。

### 4.1 評価基準
- **リズム合格条件 (`isBpmOk`)**: `medianBpm` が 100 以上、120 以下であること。
- **姿勢合格条件 (`isTiltOk`)**: `medianTilt` が 20 度以下であること。

### 4.2 総合判定とランク
合格した条件の数（`passCount`: 0〜2）に応じて3段階で判定。

1. **エキスパート級 (◎)** - `passCount === 2`
   - 「完璧な技術です！全ての基準をクリアしています。」
   - 画面上部から紙吹雪 (`canvas-confetti`) が舞う特別演出が発動。
2. **スタンダード級 (○)** - `passCount === 1`
   - 「素晴らしい技術をお持ちです。あと一歩で完璧です！」
   - いずれかの基準が満たされていないため、惜しい状態。
3. **あと一歩！ (△)** - `passCount === 0`
   - 「まずは練習に挑戦したことが素晴らしい一歩です。」
   - 初心者向けの優しいフィードバック。

### 4.3 アドバイス（上達のヒント）の動的生成
不合格となった項目がある場合のみ、該当する以下のアドバイスを表示する。
- **リズム不合格時**: 「1分間に100〜120回の一定のリズムに達していません。復習モードの音をよく聞いて、テンポを合わせる練習をしましょう。」
- **姿勢不合格時**: 「腕が少し斜めになっています。肩の真下に手を置き、真上から真っ直ぐに体重を乗せて押すことを意識しましょう。」

## 5. 設計思想とUI/UXの工夫
- **エラーハンドリング**: スマホの「戻る」ボタンや録画の異常停止によってデータが取得できなかった場合（`isAborted === true`）は、「計測を中断しました」という専用のエラーUIを Result.jsx に表示する。
- **デバッグ情報の排除**: アプリリリース版として、ユーザーの混乱を招く「座標の生データ」や「平均値・中央値などの数学用語」はUIから完全に隠蔽し、直感的な◎○△の記号と平易な日本語のみでデザインしている。
